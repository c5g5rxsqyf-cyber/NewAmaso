Security Practices

Last updated:

We take security seriously. This page describes high-level measures we use to help protect the AMASO website and communications. If you believe you’ve found a vulnerability, please email info@amaso.nl.

1) Website security

  • HTTPS/TLS (when configured) to encrypt traffic in transit.
  • Least-privilege access for servers and administration.
  • Regular dependency updates and vulnerability monitoring.
  • Basic anti-abuse protections for web forms (e.g., spam controls).

2) Data handling

  • We only request the information we need to respond to you.
  • We limit retention and restrict access to submitted messages.
  • We use reputable email/hosting providers where applicable.

3) Reporting a vulnerability

Please include a clear description, steps to reproduce, and any supporting material. Do not include sensitive personal data beyond what is necessary.

4) Responsible disclosure

We ask that you avoid privacy violations, disruption, and data destruction. We will work with you to understand and address reported issues.

5) Contact

Email: info@amaso.nl
